1) VS Code terminal settings

  • helps you keep one main terminal active
{
  "terminal.integrated.hideOnStartup": "always",
  "terminal.integrated.enablePersistentSessions": false,
  "terminal.integrated.persistentSessionReviveProcess": "never",
  "python.terminal.activateEnvironment": false,
  "debug.internalConsoleOptions": "neverOpen"
}

2) SSH to login node (environment setup chapter)

What this does:

  • connects your local workstation to the HPC login node
ssh USERNAME@skirit.metacentrum.cz

3) Request interactive session for Mamba setup

What this does:

  • asks PBS for a temporary compute node
  • reserves basic CPU/memory/time resources
qsub -I \
  -N mamba_setup \
  -l select=1:ncpus=1:mem=16gb \
  -l walltime=02:00:00 \
  -j oe

4) Load Mambaforge and create environment

What this does:

  • loads Mambaforge module
  • creates a new environment with required packages
module add mambaforge
mamba create -n geostatistics2 -c conda-forge "r-terra>=1.8"

5) Activate the environment

What this does:

  • switches shell context to your new environment
mamba activate geostatistics2

6) Install additional packages

What this does:

  • adds Python and R packages into the same environment
mamba install -c conda-forge geopandas r-sf

7) Deactivate environment

What this does:

  • returns shell to the base environment
mamba deactivate

8) SSH to login node (job submission chapter)

What this does:

  • starts a new login session to submit jobs
ssh USERNAME@skirit.metacentrum.cz

9) Define job settings variables

What this does:

  • centralizes scheduler and script parameters in one place
# Write a name for your job.
JOB_NAME="demo_job"

# Number of CPU cores requested for the job.
NCPUS=4

# Total memory requested for the job.
MEM="8gb"

# Maximum wall-clock runtime allowed for the job in hours:minutes:seconds.
WALLTIME="01:00:00"

# Name of the script that will contain the job commands.
SCRIPT_TO_RUN="run_job.sh"

# Name of the submission script passed to the HPC scheduler.
SCHEDULER_SCRIPT="submit_job.pbs"

# Name of the Mamba environment used by the job.
MAMBA_ENV="geostatistics"

# Working directory where scripts are created and the job is launched.
WORK_DIR="/storage/brno2/home/gortega/HPC-VSCode-Mamba_SciSeminar"

10) Activate environment for job preparation

What this does:

  • loads Mambaforge
  • activates the target environment
  • moves to project working directory
module add mambaforge
mamba activate "$MAMBA_ENV"
cd "$WORK_DIR"

11) Write the job payload script

What this does:

  • creates the executable script that will run on the compute node
  • embeds an example R workflow
cat > "$SCRIPT_TO_RUN" <<EOF
#!/bin/bash

set -euo pipefail

###########################################################################
# Example of R commands
############################################################################

Rscript - <<'RSCRIPT'

###########################################################################
# PASTE YOUR R CODE BELOW THIS LINE.
############################################################################



library(terra)

r <- rast(nrows = 50, ncols = 50, xmin = 0, xmax = 50, ymin = 0, ymax = 50)
values(r) <- runif(ncell(r))

mean_val <- global(r, "mean", na.rm = TRUE)
print(mean_val)



###########################################################################
# DO NOT MODIFY BELOW THIS LINE.
############################################################################

RSCRIPT

EOF

12) Make job payload executable

What this does:

  • grants execute permission to the payload script
chmod +x "$SCRIPT_TO_RUN"

13) Write PBS submission script

What this does:

  • generates a PBS script from your settings variables
  • ensures submission runs the payload in the PBS work directory
cat > "$SCHEDULER_SCRIPT" <<EOF
#!/bin/bash
#PBS -N $JOB_NAME
#PBS -l select=1:ncpus=$NCPUS:mem=$MEM
#PBS -l walltime=$WALLTIME
#PBS -j oe

cd "\$PBS_O_WORKDIR"
bash "$SCRIPT_TO_RUN"
EOF

chmod +x "$SCHEDULER_SCRIPT"

14) Submit the PBS job

What this does:

  • sends the prepared submission script to the scheduler queue
qsub "$SCHEDULER_SCRIPT"

15) Create local key for MetaCentrum VS Code

What this does:

  • creates a dedicated SSH key pair for remote development
ssh-keygen -f ~/.ssh/metacentrum_vscode

What this does:

  • keeps frontend on port 22
  • maps compute-node domains to the chosen custom port and key

  Port 26759

17) Request interactive job for VS Code server

What this does:

  • allocates a compute node where the VS Code server-side SSH daemon will run
qsub -I -N vscode_dev -l select=1:ncpus=4:mem=16gb -l walltime=04:00:00

18) Prepare SSH files on compute node

What this does:

  • creates SSH directory in scratch
  • generates temporary server key
  • prepares authorized_keys file
mkdir -p "$SCRATCHDIR/.ssh"
chmod 700 "$SCRATCHDIR/.ssh"
rm -f "$SCRATCHDIR/.ssh/server_key" "$SCRATCHDIR/.ssh/server_key.pub"
ssh-keygen -f "$SCRATCHDIR/.ssh/server_key" -N "" -C "VSCode" -q -t ed25519

touch "$SCRATCHDIR/.ssh/authorized_keys"
chmod 600 "$SCRATCHDIR/.ssh/authorized_keys"

19) Add local public key to compute node

What this does:

  • appends your local public key into authorized_keys for key-based login
cat >> "$SCRATCHDIR/.ssh/authorized_keys"
# paste one line from ~/.ssh/metacentrum_vscode.pub, then press Ctrl+D

20) Start SSH daemon on selected port

What this does:

  • verifies selected port is free
  • starts sshd on compute node
  • prints hostname for VS Code connection target
PORT=26759
netstat -tan | grep -q "0.0.0.0:$PORT " && { echo "Port $PORT is busy"; exit 1; }

/usr/sbin/sshd -D -p "$PORT" \
  -o "HostKey $SCRATCHDIR/.ssh/server_key" \
  -o "PermitRootLogin no" \
  -o "PasswordAuthentication no" \
  -o "ChallengeResponseAuthentication no" &

hostname
